#database

# База данных

PostgreSQL 16. Инициализация и миграции таблиц строго контролируются **Alembic** (`alembic/`). Хардкорные мягкие миграции удалены из кодовой базы для надежности.

## Схема связей

```
users ──< user_profiles
users ──< job_scores ──> jobs
users ──< applications ──> jobs
             applications ──< application_history
users ──< search_subscriptions
ops_events (standalone log)
```

`ix_jobs_posted_at_desc`, `ix_jobs_source`, `ix_jobs_country`, `ix_jobs_scraped_at`, `ix_jobs_country_scraped`, `ix_jobs_url_status_checked` (composite, для picker'а url-checker'а и фильтра "скрыть закрытые").

## Таблица: `ops_events`

Операционный лог системы — сканы, ошибки API, события скоринга.

| Колонка | Тип | Описание |
|---------|-----|----------|
| `id` | INTEGER PK | |
| `event_type` | VARCHAR(50) | Тип события: `scan` / `api_error` / и др. |
| `status` | VARCHAR(30) | Статус: `ok` / `error` / `warning` |
| `source` | VARCHAR(120) nullable | Источник события (имя сервиса) |
| `message` | TEXT nullable | Текстовое описание |
| `payload` | JSON nullable | Произвольные данные события |
| `created_at` | DATETIME | server default NOW() |

**Индексы:** `event_type`, `status`, `source`, `created_at`

Используется в `GET /api/ops/overview` (admin) для мониторинга состояния системы.

## Автоочистка

Вакансии старше **45 дней** удаляются ежедневно в 03:00 UTC.
Исключение: вакансии со статусом `applied` / `interviewing` / `offer` — хранятся бессрочно.
