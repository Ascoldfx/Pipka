# Тесты

Текущее покрытие — точечное, не системное. `tests/` содержит один файл с быстрыми unit-тестами на pre-filter.

## Стек

- `pytest>=8.0`
- `pytest-asyncio>=0.24`
- `ruff>=0.8` (линтер)

Все три в `[project.optional-dependencies].dev` — устанавливаются `pip install -e '.[dev]'`.

## Что покрыто

`tests/test_rules.py` — pre-filter из `app/scoring/rules.py` (см. [[Скоринг#этап-1-pre_filter--без-ai]]):

| Тест | Проверяет |
|------|-----------|
| `test_hard_reject_junior` | "Junior Procurement Analyst" → отвергается (`bucket="low"`) |
| `test_hard_reject_foreign_language` | "fluent french required" → low |
| `test_user_exclusions` | `excluded_keywords=["amazon"]` + Amazon в description → low |
| `test_english_only_filter_fail` | Профиль `english_only=True` + чисто немецкий title → low |
| `test_english_only_filter_pass` | English-friendly description → проходит |
| `test_domain_check_fail` | "Marketing Director" → low (не та функция) |
| `test_director_passes` | "Director Supply Chain" + supply chain в description → bucket="high" |
| ... | См. файл, ~15 кейсов |

Запуск:

```bash
pip install -e '.[dev]'
pytest tests/ -v
```

## Чего нет (пункты для [[Roadmap]])

- **Unit-тесты на сервисы:** `tracker_service`, `backup_service`, `user_service`, scoring backends.
- **Integration-тесты на API endpoints:** TestClient + sqlite fixture, проверка status code и shape ответа.
- **Тесты на дедуп** (`is_fuzzy_duplicate`, `_locations_conflict`).
- **Тесты на `compute_profile_hash`** — стабильность, нечувствительность к whitespace.
- **Тесты на UPSERT-логику** — что mismatched rows перезаписываются, NULL не трогаются.

## CI

`pyproject.toml` содержит конфиг ruff/pytest, но `.github/workflows/` пуст — automated runs не настроены. Тесты прогоняются только локально перед коммитом, и не всегда. См. [[Roadmap]].

```bash
# 1. Контейнер поднялся
docker logs pipka-app-1 --since 90s | grep -E 'Bot is running|Traceback|alembic'

# 2. Health endpoint жив
curl https://pipka.net/health   # → {"status":"ok","service":"pipka"}

# 3. Свежий success в ops_events
docker exec pipka-db-1 psql -U pipka -d pipka -c \
  "SELECT created_at, event_type, status FROM ops_events
   WHERE created_at > now() - interval '30 minutes'
   ORDER BY created_at DESC LIMIT 5;"
```
