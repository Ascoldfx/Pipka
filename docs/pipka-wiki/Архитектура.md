## Поток данных: Сканирование вакансий

```
APScheduler (каждые 3ч) → _background_scan()
     │
     ├─► JobAggregator.search() — параллельно, 9 источников
     │       Adzuna · JobSpy (LinkedIn/Indeed/Glassdoor) ·
     │       Arbeitnow · Remotive · Arbeitsagentur ·
     │       Xing · BerlinStartupJobs · WTTJ · Jooble
     │
     ├─► Дедупликация (exact dedup_hash + fuzzy title+company)
     │   См. [[Дедупликация]]
     │
     ├─► Bulk UPSERT в jobs (3 round-trip'а вместо N+1)
     │
     └─► Для каждого пользователя:
            ├─ pre_filter() — без AI (см. [[Скоринг#этап-1]])
            ├─ score_jobs_gemini/claude/nvidia (см. [[Скоринг#распределение]])
            ├─ UPSERT в job_scores с profile_hash/model_version
            │   (см. [[Кэш и инвалидация]])
            └─ Push в Telegram если score ≥ 80 (max 10/прогон)
```

Параллельные поллеры:
- `_backfill_score` (каждые 2ч) — оценивает накопленные unscored.
- `_nvidia_idle_rescore` (каждые 30мин) — пере-оценка DE, когда основная очередь пуста.
- `_watchlist_scan` (каждые 6ч) — точечный скан компаний из `target_companies`.
- `_cleanup_old_jobs` (03:00 UTC) — удаление вакансий >45 дней.
- `_daily_backup` (02:30 UTC) — pg_dump → B2. См. [[Бэкапы]].

## Поток данных: Авторизация

```
Browser → /auth/google/login
    → Google OAuth2 (authlib)
    → /auth/google/callback
    → get_or_create_google_user()
    → request.session["user_id"] (signed cookie pipka_session)
    → Redirect → /
```

Подробнее — [[Auth]]. CSRF на mutating-эндпоинтах — [[Безопасность#3-csrf-double-submit]].
