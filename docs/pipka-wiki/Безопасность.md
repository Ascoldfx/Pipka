#security

# Безопасность

Состоит из шести слоёв: **transport** (Cloudflare/HTTPS), **session** (signed cookie), **CSRF** (double-submit), **input validation** (Pydantic + размерные cap'ы), **DB hardening** (statement_timeout, GIN-индексы, JSONB), **AI quota cap** (rate limit).

## 1. Transport

- HTTPS-only через Cloudflare proxy → nginx → app.
- App binds на `127.0.0.1:8000` — извне напрямую недоступен.
- `SessionMiddleware`: `https_only=True, same_site="lax"` — cookie не передаётся по HTTP и не отправляется кросс-сайтом для top-level GET.
- Все ответы API получают `Cache-Control: no-store, no-cache, must-revalidate, max-age=0` (см. [[Observability#nocacheapimiddleware]]).

## 2. Session cookie

`SessionMiddleware` подписывает payload `SESSION_SECRET` (itsdangerous, см. [[Auth#сессия]]). Подмена невозможна без секрета. 30-дневный max-age. Подробнее — [[Auth]].

## 3. CSRF (double-submit)

`CSRFMiddleware` в `app/main.py` ставит на каждый запрос:

- На session lazy-генерируется `csrf_token = secrets.token_urlsafe(32)` (один раз за сессию).
- На каждый response — JS-readable cookie `csrf_token=<value>` (НЕ httponly), 30 дней.
- На POST/PUT/PATCH/DELETE требуется заголовок `X-CSRF-Token`, равный значению токена в session. `secrets.compare_digest` для constant-time сравнения.
- Несовпадение → `403 {"detail": "CSRF token missing or invalid"}`.

Исключения (`_CSRF_EXEMPT_PREFIXES`):
- `/auth/*` — Google callback приходит как top-level GET.
- `/health` — нет session, нет state-mutation.

JS-обёртка `window.fetch` в `app/static/js/app.js` автоматически читает cookie и подмешивает заголовок на unsafe-методах — call-сайты не меняются.

`/api/me` дополнительно отдаёт токен в JSON для SPA, которым неудобно читать cookie.

## 4. Input validation

### Профиль (`POST /api/profile`)

- `resume_text` — max 100 000 символов.
- `min_salary` — `0 ≤ x ≤ 1 000 000`.
- `experience_years` — `0 ≤ x ≤ 80`.
- `work_mode` — whitelist: `remote/hybrid/onsite/any`.
- `target_titles`, `preferred_countries`, `excluded_keywords`, `target_companies` — comma-split + strip; пустые элементы выкидываются.

### Resume upload (`POST /api/profile/resume`)

- Размер ≤ 10 MB (`MAX_RESUME_UPLOAD_BYTES`).
- Magic bytes verification: `.pdf` начинается с `%PDF`, `.docx` с `PK\x03\x04` — защита от extension spoofing (загрузка `.exe.pdf` не сработает).
- Whitelist форматов: PDF / DOCX / TXT.
- После парсинга — текст обрезается до `MAX_RESUME_CHARS = 100 000`, `\x00` вычищается.

### LIKE escape

В `/api/jobs?search=...` (`app/api/jobs.py`) пользовательский input идёт в `Job.title.ilike()`. Перед вставкой в pattern — escape: `\\`, `%`, `_` заменяются на свои литералы с `escape="\\"`. Не позволяет конструировать `%` для wildcards и `_` для single-char.

### SQL injection

ORM-only: `select(...).where(...)` через SQLAlchemy. Прямого `text()` с пользовательским input нет ни в одном эндпоинте. asyncpg использует prepared statements с bind-параметрами.

## 5. DB hardening

`app/database.py:_get_engine()`:

```python
connect_args["server_settings"] = {
    "statement_timeout": "30000",         # 30s per query
    "lock_timeout": "5000",               # 5s waiting on locks
    "application_name": "pipka",
}
```

Защита от runaway query (например, ILIKE по 7700 строк без индекса) — Postgres сам кикнет запрос за лимитом, коннект освободится. См. [[Деплой]] и [[Миграции]] для контекста.

GIN-индекс `ix_jobs_merged_sources` на `(raw_data->'merged_sources')` — для admin-only [[Ops панель|/api/ops/dedup]]. JSONB вместо JSON позволяет такой индекс.

## 6. AI quota cap (rate limit)

`/api/jobs/{id}/analyze` ограничен 30 запросами в час на `user_id`. Подробнее — [[Rate limiting]]. Без cap'а logged-in user мог бы за минуту выжечь дневной Gemini RPD.

## Audit-2 hotfixes (7 мая 2026, после Day-3)

Свежий проход аудита нашёл два critical-блокера прямо перед launch'ем:

- **A1 (severity 9, IDOR)** — `update_status(app_id, ...)` в `tracker_service.py` менял `Application.status` без проверки `user_id`. Telegram callback `status_<app_id>_<status>` хэндлится как `status_update_handler` без ownership-фильтра. Атака: знаешь чужую `Application.id` (легко из enumeration / shared chat) → шлёшь callback из своего Telegram → меняешь чужой статус. **Фикс:** `update_status(user_id, app_id, ...)` теперь делает `WHERE id == app_id AND user_id == user_id`. Handler передаёт `user.id` после `get_or_create_user`. Bonus — try/except ValueError на парсинг callback_data.

- **A2 (severity 8, body-OOM)** — `POST /api/profile` принимает `resume_text: str = Form(None)` через Starlette form parser, который буферит весь body в память ДО нашего `len > 100k` чека. Day-1 streaming-фикс касался только `/profile/resume` upload endpoint'а. Атакующий шлёт 500MB Form-body → OOM. **Фикс:** новый `MaxBodySizeMiddleware` проверяет `Content-Length` ≤ 12MB на всех POST/PUT/PATCH. Mounted между rate-limiter и Session, отбивает запрос ДО парсинга form-body. OAuth-callback'и exempt (Google sometimes sends large state).

## Day-3 фиксы (7 мая 2026)

Шесть medium-severity пунктов аудита (severity 4-5):

1. **#13 jsq дополнен HTML-escape** в `app/static/dashboard.html`. Раньше эскейпил только `\` и `'` — пропускал `"` (закрывает `onclick="..."`), `&`, `<`, `>`. Теперь все 6 escape'ятся; admin-таблица с user-controlled именами больше не XSS-вектор.
2. **#14 PDF/DOCX parse timeout** 30s через `asyncio.wait_for(asyncio.to_thread(_parse), timeout=30)` в `app/api/profile.py:upload_resume`. Защита от malicious PDF font-metrics inf-loop'ов и nested-XML DoS.
3. **#15 Telegram Forbidden auto-deactivate** — в `_score_and_notify` ловится `telegram.error.Forbidden`, чистится `User.telegram_id=None`, OpsEvent `telegram_forbidden`. Раньше блокировка одного user'а ронял весь push-цикл.
4. **#16 Clear session на inactive user** — `get_session_user` теперь делает `request.session.clear() + _drop_role_cache(uid)` если cookie ссылается на удалённого user'а. Без этого stale cookie жил 30 дней.
5. **#17 ON DELETE CASCADE** на 7 FK через миграцию `0005_cascade_fks.py`: `user_profiles/job_scores/applications/application_history/search_subscriptions` — все теперь cascading. `_cleanup_old_jobs` больше не зависит от ручного pre-delete порядка. Модели обновлены параллельно через `ondelete="CASCADE"`.
6. **#18 Validate job_id** в `POST /api/jobs/{id}/action`. Если job не существует → 404 cleanly, вместо FK-violation 500.

## Day-2 фиксы (6 мая 2026)

Пять high-severity пунктов аудита перед prod-релизом:

1. **TrustedHostMiddleware** outermost — отбивает forged `Host:` header'ы до того как session-state вообще аллоцируется. Allowed: `pipka.net`, `*.pipka.net`, `localhost`, `127.0.0.1`.
2. **Per-IP rate-limit middleware** (`RateLimitMiddleware` в `_ratelimit.py`). Sliding-window per-IP. Buckets: `auth-write 10/min`, `profile-write 20/min`, `api-global 300/min`. Client IP резолвится через `CF-Connecting-IP` → `X-Forwarded-For[0]` → socket. Exempt: `/static/`, `/health`, OAuth callback. См. [[Rate limiting#per-ip-middleware]].
3. **`?search=` length cap** в `/api/jobs` — `max_length=200` через `Query(...)`. Защита от 1MB substring-attack'и под `statement_timeout=30s`.
4. **Profile-list size limits** в `/api/profile`:
   - `target_titles`, `preferred_countries`, `excluded_keywords`, `target_companies` — max 50 entries, max 200 chars/entry
   - `languages` dict — max 20 keys, max 50 chars key/value
   - JSON-bomb path: проверка `isinstance(parsed, dict)` + per-key length cap
5. **Sentry PII filter** (`_sentry_before_send` в `main.py`): scrub'ает 13 ключей (resume_text, email, user_name, avatar_url, csrf_token, telegram_id, google_sub, etc.) из stack-frame locals, breadcrumb data, request headers/body, extra context. До этого `attach_stacktrace=True` + `logger.exception("update_profile failed")` на raise отгружал весь resume в Sentry.

## Day-1 фиксы перед prod-релизом (4 мая 2026)

Шесть блокеров из аудита:

1. **Stale admin role** (`require_admin_async` теперь идёт в БД с TTL 60s через `_ROLE_CACHE`). Sync-вариант помечен deprecated, оставлен для UI-cosmetics.
2. **Logout CSRF** — был GET, эксплоится через `<img src=/auth/logout>`. Теперь POST + CSRF, фронт через `fetch('POST')`. `_CSRF_EXEMPT_PREFIXES` сужен до `/auth/google/`.
3. **SecurityHeadersMiddleware** — HSTS (1 year preload), CSP, X-Frame-Options DENY, X-Content-Type nosniff, Referrer-Policy, Permissions-Policy.
4. **Resume upload streaming size check** — 64 KB chunks с running counter; `Content-Length` pre-check; OOM на 1 GB upload закрыт.
5. **ZIP bomb defense на DOCX** — `ZipInfo.file_size` проверка до `read()`, лимит 8 MB uncompressed.
6. **Session fixation** — `request.session.clear()` перед записью identity в OAuth callback.

## Известные пробелы

- **Нет 2FA** для админ-эндпоинтов — admin определяется только по email из `ADMIN_EMAILS`. Если кто-то получил доступ к Google-аккаунту, есть полный admin.
- **Нет audit log** на админские действия (`DELETE /api/admin/user/{id}`) — только в `ops_events` через middleware (api_error фильтр пропустит успешные DELETE).
- **B2 ключи** имеют write+delete права на bucket — компрометация = удаление бэкапов. План — переход на write-only application key (поддерживает Backblaze).
- **`/api/admin/user/{id}/profile`** возвращает `resume_text` — логи Sentry/uvicorn могут случайно его вытащить, если попадёт в exception. `send_default_pii=False` защищает только session, но не response payload.
