# Скоринг вакансий

Двухэтапная система: сначала быстрый rule-based фильтр, потом дорогой AI.

## Этап 1: pre_filter() — без AI

Файл: `app/scoring/rules.py`

### Жёсткие отказы (return False)
| Проверка | Причина |
|----------|---------|
| Title содержит junior/trainee/analyst/specialist/coordinator… | Слишком низкий уровень |
| Title содержит marketing/sales/HR/engineering/software/product… | Не та функция |
| Описание требует французский/испанский/польский | Кандидат не владеет |
| `excluded_keywords` из профиля пользователя | Персональные исключения |
| `english_only=True` + нет English-friendly сигналов | Только английские вакансии |
| Нет доменных слов: supply chain/procurement/operations/logistics… | Не та область |
| Зарплата ниже 70% от `min_salary` | Не проходит по деньгам |
| `plain manager` (без Director/Head/VP/Lead) + нет domain | Слишком низкий уровень |

### Бакеты
| Бакет | Условие | Идёт на AI? | Когда |
|-------|---------|-------------|-------|
| `high` | Director / VP / Head / COO / CRO / Chief + domain | ✅ | Tier 1 (приоритет) |
| `medium` | Senior Manager / Lead + domain | ✅ | Tier 1 (приоритет) |
| `manager_tier2` | Plain manager + domain (supply chain manager, ops manager…) | ✅ | Tier 2 (после Tier 1) |
| `low` | Всё остальное — junior, не та функция, иностранный язык | ❌ | Никогда (score=0 сразу) |

### Расширенные ключи для `high` бакета (22.04.2026)

Помимо стандартных Director / VP / Head / Chief / C-suite, в `DIRECTOR_KEYWORDS` также:

- **Interim / Crisis / Turnaround:** `interim manager/director/head`, `crisis manager/director`, `krisenmanager` (DE), `turnaround manager/director`, `restructuring`
- **Growth:** `growth director`
- **C-suite расширен:** добавлен `cro` (Chief Restructuring / Revenue Officer)

В `DOMAIN_KEYWORDS` для таких ролей добавлены: `crisis management`, `turnaround`, `transformation`, `restructuring`, `interim management`, `business continuity`, `operational excellence`, `continuous improvement`, `growth`.

## Этап 2: AI скоринг

### Распределение backend'ов (актуально на 23.04.2026)

| Операция | Backend | Почему |
|---------|---------|--------|
| `_score_and_notify` (реальное время, push в Telegram) | **Claude** | Откат 23.04.2026 — Gemini free tier словил 429 Rate Limit |
| `_backfill_score` (каждые 2ч) | **Gemini Flash** если `GEMINI_API_KEY` задан, иначе Claude | 4с задержка между батчами держит в лимите |
| `analyze_single_job` (кнопка «AI-анализ») | **Gemini Flash** если ключ задан, иначе Claude | 1 запрос на клик — лимит не напрягается |
| `recheck_zero_scores` (перепроверка pre-filter rejects) | **Gemini Flash** (только если ключ задан) | Дешёвая вторая проверка |

### 2a. Claude (основной — реальное время)

Файл: `app/scoring/matcher.py`

- Батч: **8 вакансий** за один запрос (`max_jobs_per_scoring_batch=8`)
- Лимит за скан: **80 вакансий** на пользователя (жёсткий cap в `_score_and_notify`)
- Кэш: **7 дней** (`score_cache_hours=168`) — не переоцениваем одно и то же
- Модель: `claude-sonnet-4-20250514` (переопределяется через `CLAUDE_MODEL` в .env)
- **Smart Retry**: экспоненциальный бэкофф на 3 попытки (`max_retries=3`) для временных таймаутов API
- Используется в: `_score_and_notify()` — реальное время, push в Telegram при score ≥ 80

### 2b. Gemini Flash (backfill + анализ, бесплатный)

Файл: `app/scoring/gemini_matcher.py`

- Активируется: если в `.env` задан `GEMINI_API_KEY`
- Модель: `gemini-2.0-flash-lite` (переопределяется через `GEMINI_MODEL`)
- Лимиты free tier: **1500 запросов/день**, 30 RPM
- Задержка между батчами: 4 секунды (`GEMINI_BATCH_DELAY`)
- Используется в: `_backfill_score()`, `analyze_single_job()`, `recheck_zero_scores()`
- **НЕ** используется в real-time скоринге (откат 23.04.2026 из-за 429 в реальном времени)
- Получение ключа: [aistudio.google.com/apikey](https://aistudio.google.com/apikey)

### Шкала оценок
| Score | Значение |
|-------|---------|
| 90–100 | Идеальное совпадение (редко!) |
| 75–89 | Сильное совпадение |
| 50–74 | Частичное совпадение |
| 30–49 | Слабое |
| 0–29 | Не подходит |

### Жёсткие штрафы в промпте
| Условие | Максимальный score |
|---------|-------------------|
| Не SC/Procurement/Ops/Logistics | 40 |
| HR/Marketing/Sales/Finance/IT | 25 |
| Plain Manager (не Director/Head/VP) | 45 |
| Требуется немецкий C1+/fluent | 30 |
| Описание полностью на немецком | 35 |
| Зарплата < 80k EUR | 35 |
| IT/Engineering навыки обязательны | 35 |
| Консалтинг/Advisory | 35 |

### english_only
Если профиль имеет `english_only=True`:
- pre_filter: отклоняет вакансии без English-friendly сигналов
- Claude промпт: получает инструкцию "вакансии на немецком/французском → max 30"

## Backfill scorer

Проблема: основной сканер скорит только вакансии из текущего скана. Если в базе уже тысячи вакансий — большинство остаются без оценки.

Решение: `_backfill_score()` каждые **2 часа** берёт все вакансии за `job_max_age_days` дней и обрабатывает двухуровневой очередью:
- `low` bucket → `JobScore(score=0)` мгновенно, без AI-вызовов (дренаж очереди)
- `high`/`medium` → AI-скоринг, до 1000 за запуск (Tier 1, приоритет)
- `manager_tier2` → AI-скоринг, до 1000 за запуск (Tier 2, только когда Tier 1 пуст)

**AI backend для backfill** выбирается автоматически:
- `GEMINI_API_KEY` задан → **Gemini Flash** (бесплатно)
- `GEMINI_API_KEY` пустой → **Claude** (платно)
