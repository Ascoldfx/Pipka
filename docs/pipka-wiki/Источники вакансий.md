# Источники вакансий

Все источники реализуют интерфейс `JobSource` (`app/sources/base.py`).
Запускаются параллельно через `asyncio.gather()`.

## Adzuna (`app/sources/adzuna.py`)
- **API:** `https://api.adzuna.com/v1/api/jobs`
- **Страны:** de, at, nl, ch, be, si, sk, ro, hu (и другие из профиля)
- **Лимит:** 50 вакансий на запрос, несколько страниц
- **Особенности:** даёт зарплаты, хорошее покрытие DACH

## JobSpy (`app/sources/jobspy_source.py`) — LinkedIn, Indeed, Glassdoor
- **Библиотека:** `python-jobspy`
- **Источники внутри:** LinkedIn, Indeed, Glassdoor (через scraping)
- **Особенности:** LinkedIn игнорирует параметр `country_indeed` и возвращает глобальные результаты
- **Фикс (апрель 2026):** для LinkedIn `country=None` в `RawJob` → принудительно запускается текстовый фильтр `_is_wrong_location()`. Для Indeed/Glassdoor `country=country.upper()` — они корректно соблюдают фильтрацию по стране.
- **Защита:** `_is_wrong_location()` жёстко блокирует `country in {us, ca}` + US-keywords в тексте + "greater [city]" шаблоны

## Arbeitnow (`app/sources/arbeitnow.py`)
- **API:** `https://www.arbeitnow.com/api/job-board-api`
- **Бесплатный публичный API**
- **Фокус:** международные вакансии в Германии, много English-speaking

## Remotive (`app/sources/remotive.py`)
- **API:** `https://remotive.com/api/remote-jobs`
- **Фокус:** remote-only вакансии
- **Особенности:** много US компаний → `_is_wrong_location()` фильтрует

## Arbeitsagentur (`app/sources/arbeitsagentur.py`)
- **API:** `https://rest.arbeitsagentur.de/jobboerse/jobsuche-service/pc/v4/jobs`
- **Страны:** только `de`
- **Auth:** API-ключ в заголовке `X-API-Key` (`ARBEITSAGENTUR_API_KEY`)
- **Особенности:** официальный портал занятости Германии. Распознаёт `HOME_OFFICE` в `arbeitszeitmodelle` → `is_remote=True`

## Xing (`app/sources/xing.py`)
- **URL:** `https://www.xing.com/jobs/search`
- **Метод:** Scraping SSR HTML — парсит `window.crate.serverData.APOLLO_STATE` (JSON)
- **Страны:** DACH + BE, NL, PL, CZ, GB, FR (`xing_countries`)
- **Лимит:** ~20 результатов на запрос (Xing ограничивает неавторизованных)
- **Fallback:** если `APOLLO_STATE` не парсится — regex-парсинг `VisibleJob:*` объектов
- **Особенности:** нет описания вакансии (требует авторизации); title + company используются для скоринга

## BerlinStartupJobs (`app/sources/berlinstartupjobs.py`)
- **URL:** RSS-фиды `https://berlinstartupjobs.com/operations/feed/` и `.../finance/feed/`
- **Метод:** Парсинг RSS (stdlib `xml.etree.ElementTree`) + fetch полной страницы вакансии
- **Фокус:** Operations, Supply Chain, Finance в берлинских стартапах; англоязычные вакансии
- **Фильтр по заголовку:** `TITLE_KEYWORDS` — operations, procurement, logistics, COO, director, head of, sourcing, warehouse, fulfillment и др.
- **Особенности:**
  - RSS даёт только ~200 символов описания → для релевантных вакансий делает дополнительный GET на страницу вакансии
  - Формат заголовка в RSS: `"Job Title // Company Name"` — парсится `_extract_company()`
  - `country="DE"`, `location="Berlin, Germany"` для всех вакансий
- **Добавлен:** апрель 2026

## WTTJ — Welcome to the Jungle / Otta (`app/sources/wttj.py`)
- **API:** Algolia search index `wttj_jobs_production_en`
- **Endpoint:** `POST https://csekhvms53-dsn.algolia.net/1/indexes/*/queries`
- **Auth:** публичные client-side ключи (`x-algolia-application-id: CSEKHVMS53`), требуют `origin/referer: https://www.welcometothejungle.com`
- **Фокус:** стартапы по всей Европе; operations, procurement, finance, sales
- **Фильтр стран:** `facetFilters: [["offices.country_code:DE", ...]]` — из `params.countries`
- **Описание:** `summary` + `key_missions` (список) + `profile` (markdown) → объединяется до 4000 символов
- **Зарплата:** возвращает `salary_minimum/maximum/currency` — одна из немногих площадок с реальными данными
- **URL вакансии:** `https://www.welcometothejungle.com/en/companies/{org_slug}/jobs/{slug}`
- **Добавлен:** апрель 2026 (ранее назывался Otta, куплен WTTJ в январе 2024)

## Watchlist (`app/sources/watchlist.py`)
- **Источник данных:** Adzuna API (компанийный фильтр `company=`)
- **Цель:** поиск вакансий в конкретных компаниях из `target_companies` профиля
- **Запросы:** `params.queries` — список компаний (не поисковых фраз)
- **source в БД:** `watchlist`
- **Особенности:** ищет до 50 вакансий на компанию по всем странам из профиля

## Фильтрация при агрегации

```
Все сырые вакансии
    │
    ├─ Дедупликация по dedup_hash (SHA-256 от title+company) — Pass 1
    │
    ├─ Нечёткая дедупликация is_fuzzy_duplicate() — Pass 2
    │       ├─ _normalize(title) совпадает точно
    │       ├─ _are_same_company(): токен-сабсет (≥4 символа) или префикс для коротких (BMW)
    │       └─ _locations_conflict(): оба unknown → разрешить; асимметрия → запретить; разные города → запретить
    │   raw_data["merged_sources"] = список источников объединённой вакансии
    │
    ├─ _is_negative() — отклоняет junior/wrong function/German C1
    │
    ├─ Возраст > max_age_days — отклоняет устаревшие
    │
    └─ _is_wrong_location() — отклоняет не-европейские:
            ├─ job.country in ALLOWED_COUNTRIES → сразу разрешаем
            ├─ job.country in {us, ca} → сразу отказ
            ├─ Location содержит DACH/EU маркер → разрешаем
            ├─ Location содержит US штат/город → отказ
            ├─ Location содержит Canadian город → отказ
            ├─ Description содержит US keywords → отказ (если нет EU маркера)
            └─ ALLOWED_COUNTRIES: de, at, nl, ch, be, si, sk, ro, hu, pl, cz, fr, es, it, pt, gb, uk, ie, dk, se, no, fi, lu
```

## Параметры поиска

Динамически формируются из профилей пользователей:
- **Запросы:** объединение `target_titles` всех пользователей
- **Страны:** объединение `preferred_countries` всех пользователей
- **Фолбэк:** дефолтные `SCAN_QUERIES` и страны DACH+ если нет профилей
