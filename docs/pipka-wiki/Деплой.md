# Деплой

## Окружение

| Параметр | Значение |
|----------|---------|
| Сервер | Contabo VPS, Ubuntu 24.04 |
| SSH | `ssh root@217.76.61.28 -i ~/.ssh/id_ed25519` |
| Каталог | `/opt/pipka` |
| Домен | `pipka.net` через Cloudflare proxy |
| Стек | Docker Compose: `pipka-app-1` + `pipka-db-1` |

## Контейнеры

`docker-compose.yml`:

- **app** — FastAPI + Telegram bot + APScheduler в одном процессе. Порт `127.0.0.1:8000` (за nginx). Volume `app_data` под `/app/data` (бэкапы).
- **db** — `postgres:16-alpine`. Volume `pgdata` под `/var/lib/postgresql/data`. Healthcheck `pg_isready -U pipka` каждые 5с.

App ждёт healthy db (`depends_on.db.condition: service_healthy`). Логи каждого контейнера ротируются по `10MB × 3` файла (json-file driver).

## Стандартный цикл деплоя

```bash
# Локально
git add <files> && git commit -m "..." && git push origin main

# На сервере
ssh root@217.76.61.28 -i ~/.ssh/id_ed25519
cd /opt/pipka && git pull && docker compose up -d --build
```

`--build` обязателен — без него Docker подцепит закэшированный image и код не обновится.

## Что происходит на старте контейнера

1. `init_db()` (см. `app/database.py`) запускает `alembic upgrade head` через `run_in_executor`. Подробнее — [[Миграции]].
2. FastAPI поднимается на `0.0.0.0:8000`.
3. Telegram bot стартует long-polling к Telegram API.
4. APScheduler регистрирует jobs (см. [[Сервисы#scheduler]]) и запускает первый `startup_scan` через 30с.
5. Если задан `SENTRY_DSN` — инициализируется SDK ДО создания FastAPI app (см. [[Observability#sentry]]).

## Конфигурация

Все настройки в `/opt/pipka/.env`. Полный список — [[Настройки]]. Минимальный набор для запуска:

```env
TELEGRAM_BOT_TOKEN=...
ADZUNA_APP_ID=...
ADZUNA_APP_KEY=...
ANTHROPIC_API_KEY=...
GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...
SESSION_SECRET=<32+ random bytes>
POSTGRES_PASSWORD=...
```

Опционально включаются: `GEMINI_API_KEY`, `NVIDIA_API_KEY`, `B2_KEY_ID/B2_APP_KEY/B2_BUCKET`, `SENTRY_DSN`. Каждый — лениво активируется при заполнении (см. [[Скоринг]], [[Бэкапы]], [[Observability]]).

## Откат

```bash
ssh root@217.76.61.28 -i ~/.ssh/id_ed25519
cd /opt/pipka
git log --oneline -10           # выбрать предыдущий зелёный коммит
git checkout <sha>
docker compose up -d --build
```

Миграции БД не откатываются автоматически: Alembic `downgrade` не вызывается. Если нужно — вручную `alembic downgrade -1` в контейнере. См. [[Миграции]].

## Проверка после деплоя

```bash
ssh root@217.76.61.28 -i ~/.ssh/id_ed25519
docker logs pipka-app-1 --since 2m | grep -E 'Bot is running|Traceback|alembic'
curl https://pipka.net/health
docker exec pipka-db-1 psql -U pipka -d pipka -c \
  "SELECT created_at, event_type, status, left(message,80) FROM ops_events \
   WHERE created_at > now() - interval '15 minutes' ORDER BY created_at DESC LIMIT 5;"
```

Триада: бут → health 200 → свежий `scan success` в [[Ops панель|ops_events]].
