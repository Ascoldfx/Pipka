## Важные особенности

### Docker COPY . .
Файлы бекаются при `--build`. Без `--build` — старый код.
**Всегда:** `docker compose up -d --build`

### Миграции БД
Автоматические soft migrations при каждом старте в `app/database.py`.
Каждая в отдельной транзакции (`IF NOT EXISTS`).

### .env на сервере
Не хранится в git. Редактировать напрямую:
```bash
nano /opt/pipka/.env
docker compose up -d --build
```

### Cloudflare
pipka.net проксируется через Cloudflare.
`dig pipka.net` → CloudFlare IP.
Реальный IP: `217.76.61.28`.

## Полезные команды

```bash
# Бекап БД
docker compose exec db pg_dump -U pipka pipka > backup_$(date +%Y%m%d).sql

# Восстановление
docker compose exec -T db psql -U pipka pipka < backup_20260416.sql

# Войти в БД
docker compose exec db psql -U pipka pipka

# Кол-во вакансий по источникам
docker compose exec db psql -U pipka pipka -c "SELECT source, COUNT(*) FROM jobs GROUP BY source ORDER BY count DESC;"

# Статусы заявок
docker compose exec db psql -U pipka pipka -c "SELECT status, COUNT(*) FROM applications GROUP BY status;"
```
