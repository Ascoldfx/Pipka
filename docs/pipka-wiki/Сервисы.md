## Scheduler

Файл: `app/services/scheduler_service.py`

| Job ID | Тип | Расписание | Функция |
|--------|-----|-----------|---------|
| `startup_scan` | date | +30с после старта | Первый скан при запуске |
| `background_scan` | interval | каждые 3 часа | Основной сбор вакансий |
| `backfill_score` | interval | каждые **2 часа** | Скоринг накопленных вакансий (двухуровневый) |
| `nvidia_idle_rescore` | interval | каждые **30 мин** | NVIDIA (Gemma) пересмотр вакансий DE когда очередь Gemini пуста |
| `daily_cleanup` | cron | 03:00 UTC | Удаление старых вакансий |
| `daily_backup` | cron | **02:30 UTC** | pg_dump → gzip → local + B2 |
| `watchlist_scan` | interval | каждые 6 часов | Поиск по компаниям из `target_companies` |
| `check_job_urls` | cron | **04:00 UTC** | HEAD-пинг по `Job.url` для пометки закрытых, drain-loop до пустой очереди ([[Проверка ссылок]]) |
| `embed_index` | interval | каждые **2 часа** | Заполнение `jobs.embedding` / `user_profiles.embedding` через Gemini embedding API ([[Поиск и индексация]]) |
| `embed_index_startup` | date | +90с после старта | Первый прогон индексатора при запуске |

### _background_scan()
1. Загружает всех активных пользователей с профилями
2. Собирает динамические запросы из `target_titles` всех пользователей
3. Запускает `JobAggregator.search()` — параллельно по **9 источникам**: Adzuna, JobSpy (LinkedIn/Indeed/Glassdoor), Arbeitnow, Remotive, Arbeitsagentur, Xing, BerlinStartupJobs, WTTJ, Jooble
4. Для каждого пользователя → `_score_and_notify()`

### _score_and_notify()
1. Определяет уже оценённые вакансии
2. Исключает hidden (applied/rejected) через [[Трекер]]
3. Прогоняет через `pre_filter()`
4. Скорит через `score_jobs()` (max 80)
5. Push в Telegram все `score >= 80` (max 10 сообщений)

### _backfill_score()
Двухуровневая очередь (апрель 2026):
1. Берёт все вакансии за `job_max_age_days` дней
2. Исключает уже оценённые и hidden
3. Прогоняет через `pre_filter()`:
   - **Жёсткий отказ** (`low`) → сразу пишет `JobScore(score=0)` — без Claude, очередь дренируется мгновенно
   - **Tier 1** (`high`/`medium`) — director/VP/head + domain → AI-скоринг, до **1000** за запуск
   - **Tier 2** (`manager_tier2`) — plain manager + domain → AI-скоринг только когда Tier 1 пуст, до **1000** за запуск
4. Запускается каждые **2 часа**

### _cleanup_old_jobs()
1. Находит вакансии с `applied/interviewing/offer` → защищает
2. Удаляет остальные старше `JOB_MAX_AGE_DAYS` (45 дней)
3. Удаляет батчами по 500: сначала `job_scores`, потом `applications`, потом `jobs`

## Трекер

Файл: `app/services/tracker_service.py`

| Функция | Действие |
|---------|---------|
| `mark_applied(user_id, job_id)` | Статус `applied` |
| `mark_rejected(user_id, job_id)` | Статус `rejected` |
| `save_job(user_id, job_id)` | Статус `saved` |
| `get_hidden_job_ids(user_id)` | applied + rejected IDs |
| `get_hidden_dedup_hashes(user_id)` | dedup_hash скрытых вакансий |
| `check_auto_exclude_company(user_id, job_id)` | Автоисключение компании после 5 rejections |

### check_auto_exclude_company()
Вызывается из `/api/jobs/{id}/action?action=reject` после `mark_rejected()`.

1. Загружает job → берёт `company_name`
2. Считает кол-во rejected-приложений от этой компании (case-insensitive)
3. Если `count >= 5` и компания ещё не в `excluded_keywords`:
   - Добавляет в `profile.excluded_keywords`
   - Обнуляет `JobScore.score=0` для всех незатронутых вакансий этой компании
4. Возвращает `company_name` (str) или `None`
5. Frontend показывает toast-уведомление при `auto_excluded != null`

Константа: `AUTO_EXCLUDE_THRESHOLD = 5`

## Бэкап БД

Файл: `app/services/backup_service.py`

### run_backup()
1. Пропускает если `DATABASE_URL` не PostgreSQL
2. Создаёт `/app/data/backups/` (внутри `app_data` volume)
3. `pg_dump -h HOST -p PORT -U USER DBNAME` через `subprocess.run`
4. `PGPASSWORD` передаётся через env, не в CLI
5. Gzip → `pipka_YYYYMMDD_HHMMSS.sql.gz`
6. Ротация: удаляет старые, оставляет последние **7**
7. Если `B2_KEY_ID/B2_APP_KEY/B2_BUCKET` заданы → загружает в B2 (best-effort)

**Требования Docker:** `postgresql-client` в Dockerfile
**Зависимость Python:** `boto3>=1.34` (только для B2, опционально)

## Агрегатор

Файл: `app/sources/aggregator.py`

### Фильтры при сборе

**_is_negative()** — отклоняет по title/description:
- `NEGATIVE_KEYWORDS`: ausbildung, junior, trainee, specialist, analyst…
- `EXCLUSION_PHRASES`: только для инвалидов, специальные программы
- `GERMAN_C1_REQUIRED`: fließend deutsch, verhandlungssicher, sehr gute Deutschkenntnisse…
- `FOREIGN_LANG_REQUIRED`: français requis, spanish required…

**_is_wrong_location()** — отклоняет не-DACH вакансии:
- Жёсткий блок: `job.country in {"us", "ca"}` → сразу отказ
- Проверка по location: US штаты, US города, Canadian города/провинции
- Проверка по description: US_KEYWORDS, NON_DACH_CITIES, NON_DACH_RUSSIAN
- Проверка DACH: если есть Germany/Berlin/Amsterdam/Wien… → разрешаем
