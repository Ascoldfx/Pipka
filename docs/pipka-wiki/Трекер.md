# Трекер заявок

Файл: `app/services/tracker_service.py`. Управляет статусами вакансий пользователя через таблицы [[База данных#applications]] и `application_history`.

## Допустимые статусы

`saved → applied → interviewing → offer | rejected | withdrawn`

Любая смена статуса фиксируется в `application_history` (старый → новый + timestamp) — для аудита и метрик.

## Публичные функции

| Функция | Что делает |
|---------|-----------|
| `save_job(user_id, job_id, session)` | Создаёт `Application(status='saved')`, idempotent. |
| `mark_applied(user_id, job_id, session)` | `status='applied'`, `applied_at=now()`. Пишет history. |
| `mark_rejected(user_id, job_id, session)` | `status='rejected'`. Пишет history. Триггерит [[#Auto-exclude]]. |
| `update_status(user_id, job_id, status, session)` | Универсальный апдейт + history. Валидирует против `VALID_STATUSES`. |
| `get_hidden_job_ids(user_id, session)` | `applied + rejected` IDs — скрываются из inbox/scan. |
| `get_hidden_dedup_hashes(user_id, session)` | Те же скрытия, но по `Job.dedup_hash` — ловит свежие реинкарнации той же вакансии из разных источников. |
| `check_auto_exclude_company(user_id, job_id, session)` | См. ниже. |

## Auto-exclude

Константа `AUTO_EXCLUDE_THRESHOLD = 5`.

Вызывается из `mark_rejected` через handler `/api/jobs/{id}/action?action=reject` (см. [[API#jobs]]):

1. Берём `Job.company_name`.
2. Считаем все `Application.status='rejected'` от этой компании за всё время (case-insensitive).
3. Если `count >= 5` И компания ещё не в `profile.excluded_keywords`:
   - Добавляем `company_name` в массив `excluded_keywords`.
   - Bulk UPDATE `JobScore.score=0` всех вакансий этой компании у пользователя — они уйдут из топ-листа сразу.
4. Возвращаем `company_name` (str) или `None`.
5. Frontend показывает toast "Компания X автоматически исключена" при `auto_excluded != null` в ответе.

Так пользователь, который методично отказывается от одной компании, перестаёт видеть её предложения без ручной правки фильтров.

## Связи

- Дёргается из [[API#jobs-actions]] (`/api/jobs/{id}/action`).
- `get_hidden_*` потребляется [[Сервисы#scheduler]] в `_score_and_notify` и `_backfill_score` — чтобы не пере-оценивать уже-обработанные.
- Записи в `application_history` читает [[Ops панель]] для отображения активности пользователя.
