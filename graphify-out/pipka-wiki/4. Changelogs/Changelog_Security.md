# Feature Release: HTTP Basic Authentication
**Дата:** 14 Апреля 2026
**Связанные ветки/Домены:** [[config.md]], [[routes.md]]

---

## 📌 Описание Задачи
Возникла критическая уязвимость: веб-дашборд и API `pipka.net` были открыты для публичного доступа. Любой пользователь, знающий IP или домен сервера, мог получить доступ к списку вакансий, изменять настройки (в том числе `excluded_keywords`) и запускать новые сканирования, тем самым сжигая API-кредиты Claude.

## 🔒 Решение (HTTP Basic Auth)
Внедрена классическая браузерная аутентификация, которая полностью блокирует доступ ко всем маршрутам и статическим файлам, пока не введен правильный логин и пароль.

### 1. Глобальная блокировка Router'a (`app/api/dashboard.py`)
Мы использовали возможности `FastAPI Security`.
Создан метод `verify_credentials()`, использующий безопасную проверку токенов (`secrets.compare_digest`), чтобы избежать side-channel атак (тайминговых атак).
Далее этот `Depends` был наложен не на отдельные эндпоинты, а глобально на весь `APIRouter()` — что на 100% гарантирует, что ни один новый маршрут случайно не "утечет" в публичный доступ в будущем.

### 2. Ключи Доступа (`app/config.py`)
По просьбе администратора, дефолтные credentials были жестко зашиты в конфигурацию (как `ascoldfx` и приватный пароль) вместо `admin` / `pipka123`.

*Примечание:* Данные дефолтные параметры могут быть переопределены серверным файлом `.env` без переписывания исходного кода, что является лучшей практикой 12-Factor App инфраструктур.
