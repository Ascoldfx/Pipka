## 🚀 Деплой и Синхронизация (Local -> Remote)
Вся разработка ведется на локальном Mac (`~/клод джоб/Pipka/`). Приложение не использует традиционный CI/CD пайплайн (GitHub Actions). Код переносится напрямую на сервер утилитой `rsync`.

### Команда Синхронизации
Запускается на локальном Mac (Termianl) для безопасной затирки сервера новым кодом:
```bash
rsync -avz --exclude='venv' --exclude='__pycache__' --exclude='alembic' --exclude='.git' ~/клод\ джоб/Pipka/ root@217.76.61.28:/opt/jobhunt/
```
*Внимание:* Папка `alembic` и кэш исключаются, так как база данных живет собственной жизнью на сервере.

## 🐳 Управление Контейнерами (Docker)
Проект полностью изолирован с помощью `Docker Compose`. После выполнения синхронизации `rsync`, необходимо пересобрать контейнер на сервере с новым кодом.

1. Зайти на сервер по SSH:
```bash
ssh root@217.76.61.28
```
2. Перейти в директорию:
```bash
cd /opt/jobhunt
```
3. Пересобрать и перезапустить приложение (в тихом фоне):
```bash
docker compose up -d --build
```
4. Посмотреть логи бота/агрегатора при падении:
```bash
docker compose logs -f app
```

## 🔐 Переменные Окружения (`.env`)
Для запуска проекта в корне директории (`Pipka/` или `/opt/jobhunt/`) **обязан** существовать файл `.env`. Без этих ключей проект крашнется:

| Ключ | Назначение | Статус |
| :--- | :--- | :--- |
| `DATABASE_URL` | Путь подключения к PostgreSQL или SQLite. | Critical |
| `ANTHROPIC_API_KEY` | Доступ к Claude 3 (Скорринг, Аналитика вакансий). | Critical |
| `TELEGRAM_BOT_TOKEN` | Токен из BotFather для Telegram UI. | Critical |
| `ADZUNA_APP_ID` | API для агрегатора Adzuna Source. | Optional (Scraper) |
| `ADZUNA_APP_KEY` | Секрет агрегатора Adzuna Source. | Optional (Scraper) |
| `ADMIN_TELEGRAM_IDS` | JSON массив ID пользователей с правами админа. | Critical |
| `DASHBOARD_USERNAME` | Логин для входа в веб-панель (по умолчанию admin). | Critical (Security) |
| `DASHBOARD_PASSWORD` | Пароль для входа в веб-панель (по умолчанию pipka123). | Critical (Security) |

*Важно:* Файл `.env` никогда не должен добавляться в кодовую базу (gitignore), он настраивается вручную на продакшн-сервере.
